## Overview

[GitHub Copilot](https://github.com/features/copilot) is an AI-powered coding assistant developed by **GitHub** (a Microsoft subsidiary). It provides code completions, chat-based assistance, multi-file editing, autonomous agent capabilities, and code review -- integrated across VS Code, Visual Studio, JetBrains IDEs, Xcode, Eclipse, GitHub.com, GitHub Mobile, and the CLI. Originally launched in 2021 as an inline code suggestion tool, Copilot has evolved into a comprehensive agentic development platform with support for multiple AI models, MCP servers, custom agents, skills, and a cloud-hosted coding agent.

Copilot is available in several tiers: **Copilot Free** (limited usage), **Copilot Pro**, **Copilot Pro+**, **Copilot Business**, and **Copilot Enterprise**. Higher tiers unlock additional models, higher usage quotas, and enterprise governance features.

<Info>
  GitHub Copilot is multi-model. It supports models from OpenAI (GPT-4.1, GPT-5, o4-mini), Anthropic (Claude Sonnet 4.5, Claude Opus 4.1), Google (Gemini 2.5 Pro, Gemini 2.0 Flash), xAI (Grok), and others. An **Auto** mode automatically selects the optimal model per request. Users can also bring their own API keys for additional models.
</Info>

### Key Features

| Feature | Description |
|---|---|
| **Code Completions** | Inline suggestions as you type, including multi-line and full-function completions |
| **Copilot Chat** | Conversational AI assistant with chat participants (`@workspace`, `@terminal`, `@vscode`) and chat variables |
| **Copilot Edits** | Multi-file editing mode with a working set UI for iterative code changes |
| **Agent Mode** | Autonomous coding agent in the IDE that plans, edits files, runs terminal commands, and self-corrects |
| **Coding Agent** | Cloud-hosted agent on GitHub.com that works from issues, creates branches, and opens pull requests |
| **Code Review** | AI-powered pull request review on GitHub.com with customizable review instructions |
| **MCP Support** | Extends Copilot with external tools and data sources via the Model Context Protocol |
| **Custom Instructions** | Repository-level and path-specific instruction files that guide Copilot behavior |
| **Custom Agents** | Specialized agent personas defined in `.github/agents/` with scoped tools and instructions |
| **Agent Skills** | Portable capability packages with instructions, scripts, and resources loaded on demand |
| **Hooks** | Deterministic shell commands that execute at key points in the agent workflow |
| **Multi-Model** | Choose from 15+ models or use Auto mode for optimal model selection |

## Configuration

### Custom Instructions

GitHub Copilot supports a layered instruction system with multiple file types at different scopes.

<Tabs>
  <Tab title="Repository Instructions">
    The primary instruction file is `.github/copilot-instructions.md` in the repository root. Its contents are automatically included in all Copilot Chat requests within the workspace.

    ```markdown .github/copilot-instructions.md
    This is a TypeScript monorepo using pnpm workspaces.

    - Use strict TypeScript with no `any` types
    - Prefer functional React components with hooks
    - Use date-fns instead of moment.js (moment is deprecated)
    - All API routes must validate input with Zod schemas
    - Write tests using Vitest, not Jest
    ```

    <Note>
      Custom instructions are enabled by default. You can verify they are being used by checking the **References** list in a Copilot Chat response -- the instruction file will appear as a reference.
    </Note>
  </Tab>
  <Tab title="Path-Specific Instructions">
    Create `*.instructions.md` files in `.github/instructions/` with YAML frontmatter specifying an `applyTo` glob pattern. These instructions apply only to matching files.

    ```markdown .github/instructions/frontend.instructions.md
    ---
    applyTo: "src/components/**/*.tsx"
    ---
    - Use Tailwind CSS utility classes, never inline styles
    - All components must export a Props type
    - Prefer composition over inheritance
    ```

    ```markdown .github/instructions/api.instructions.md
    ---
    applyTo: "src/api/**/*.ts"
    excludeAgent: "code-review"
    ---
    - All endpoints must include OpenAPI documentation
    - Use middleware for authentication checks
    - Return consistent error response shapes
    ```

    The `excludeAgent` property controls which Copilot agents use a given instructions file. Set `excludeAgent: "code-review"` to hide instructions from Copilot code review, or `excludeAgent: "copilot-coding-agent"` to exclude them from the coding agent.
  </Tab>
  <Tab title="Personal / Global Instructions">
    Personal instructions apply across all repositories for your own usage:

    | Location | Scope |
    |---|---|
    | `$HOME/.copilot/copilot-instructions.md` | Personal global instructions |
    | `COPILOT_CUSTOM_INSTRUCTIONS_DIRS` env var | Comma-separated list of directories containing instruction files |

    Personal instructions take the highest priority, followed by repository instructions, then organization instructions.
  </Tab>
  <Tab title="AGENTS.md">
    Copilot also reads `AGENTS.md` files for cross-agent compatibility. If both `AGENTS.md` and `.github/copilot-instructions.md` exist at the repository root, instructions from both files are used.

    Copilot also reads `CLAUDE.md` and `GEMINI.md` files when present, providing compatibility with other AI agent instruction formats.
  </Tab>
</Tabs>

### File Locations

<Tabs>
  <Tab title="Repository Files">
    | File / Directory | Purpose |
    |---|---|
    | `.github/copilot-instructions.md` | Repository-wide custom instructions for Copilot |
    | `.github/instructions/*.instructions.md` | Path-specific instructions with `applyTo` globs |

    | `.github/skills/[name]/SKILL.md` | Agent skill packages |

    | `.vscode/mcp.json` | MCP server configuration (workspace-scoped) |
    | `.vscode/settings.json` | VS Code Copilot settings (workspace-scoped) |
  </Tab>
  <Tab title="User-Level Files">
    | File / Directory | Purpose |
    |---|---|
    | `$HOME/.copilot/copilot-instructions.md` | Personal global instructions |

  </Tab>
</Tabs>

## Copilot Chat

Copilot Chat provides a conversational interface for asking questions, generating code, and performing tasks. It operates in three modes:

| Mode | Description |
|---|---|
| **Ask** | Traditional chat -- answers questions without modifying files |
| **Edit** | Suggests and applies edits directly to files in a working set |
| **Agent** | Autonomous mode -- plans, edits files, runs terminal commands, and self-corrects |

### Chat Participants

Chat participants are domain experts invoked with `@` mentions:

| Participant | Description |
|---|---|
| `@workspace` | Has context about the entire codebase; answers project-wide questions |
| `@terminal` | Knows about the integrated terminal, its contents, and command history |
| `@vscode` | Knows about VS Code editor commands, settings, and features |
| Custom agents | Third-party and user-defined agents invoked by name (e.g., `@test-agent`) |

### Chat Variables

Chat variables provide specific context using `#` syntax:

| Variable | Description |
|---|---|
| `#file` | Reference a specific file in the workspace |
| `#selection` | The currently selected code in the editor |
| `#editor` | The visible content in the active editor |
| `#codebase` | The full codebase context |
| `#git` | Git history and changes |
| `#terminalLastCommand` | The last command run in the terminal |
| `#websearch` | Web search results (requires Web Search extension) |

### Slash Commands

| Command | Description |
|---|---|
| `/fix` | Fix errors in the active file or selected code |
| `/tests` | Generate tests for the active file or selection |
| `/explain` | Explain selected code |
| `/help` | Show available commands and features |
| `/yolo` or `/autoApprove` | Enable auto-approval for tool invocations |
| `/disableYolo` or `/disableAutoApprove` | Disable auto-approval |
| `/skills` | Open the Configure Skills menu |

Custom slash commands can be created using `.prompt.md` files that define reusable prompts and can reference specific agents and MCP tools.

## Agent Mode

Agent mode is Copilot's autonomous coding capability in the IDE. It goes beyond chat and edits by independently planning and executing multi-step tasks.

### How It Works

<Steps>
  <Step title="Task Analysis">
    Copilot analyzes your prompt and determines which files, tools, and steps are needed to complete the task.
  </Step>
  <Step title="Autonomous Execution">
    The agent iterates through a loop: reading files, proposing code changes, running terminal commands (e.g., installing packages, running tests, compiling), and monitoring output.
  </Step>
  <Step title="Self-Correction">
    If compile errors, lint errors, or test failures occur, the agent automatically attempts to fix them and re-runs verification.
  </Step>
  <Step title="Review">
    All changes are presented for your review. You can accept, reject, or request modifications before finalizing.
  </Step>
</Steps>

### Built-in Tools

Agent mode uses a set of built-in tools that can be extended with MCP servers:

| Tool Category | Capabilities |
|---|---|
| **File Operations** | Search workspace, read files, create files, apply edits |
| **Terminal** | Run shell commands, install packages, execute tests |
| **Editor Integration** | Get compile/lint errors, access editor state |
| **Code Search** | Semantic and text-based search across the codebase |

### Permission Levels

The permissions picker in the Chat view controls agent autonomy:

| Level | Behavior |
|---|---|
| **Ask** | Every tool invocation requires explicit approval |
| **Auto-approve reads** | File reads are automatic; writes and terminal commands need approval |
| **Auto-approve all** | Full autonomy (equivalent to `/yolo` mode) |

## Coding Agent (GitHub.com)

The Copilot coding agent is a cloud-hosted autonomous agent that operates directly on GitHub. It works like a virtual team member -- you assign it an issue and it delivers a pull request.

### How It Works

1. **Assign an issue** to Copilot on GitHub.com, GitHub Mobile, GitHub CLI, or from VS Code
2. The agent spins up a **secure development environment** powered by GitHub Actions
3. It creates a branch, makes changes, pushes commits to a **draft pull request**
4. It **self-reviews** using Copilot code review before opening the PR
5. It runs **security scanning** (code scanning, secret scanning, dependency checks)
6. You review the PR and merge when satisfied

### Task Sources

Issues can be assigned from GitHub.com, GitHub Mobile, GitHub CLI, VS Code, Slack, Teams, Azure Boards, Jira, Raycast, and Linear.

### Custom Agents for Coding Agent

Define specialized agents in `.github/agents/`:

```markdown .github/agents/test-agent.md
---
name: "test-agent"
description: "Specialized agent for writing and improving tests"
---

You are a test engineering specialist. When assigned an issue:

1. Analyze the code under test
2. Write comprehensive unit and integration tests
3. Ensure edge cases are covered
4. Run the test suite and verify all tests pass
5. Report coverage metrics in the PR description
```

### Model Selection

The coding agent supports model selection per task. Available models include Claude Sonnet 4.5, Claude Opus 4.5, Claude Opus 4.6, GPT-5.1-Codex, GPT-5.1-Codex-Max, and GPT-5.2-Codex.

<Warning>
  Tasks assigned to the Copilot coding agent consume both GitHub Actions minutes and premium requests from your Copilot subscription quota.
</Warning>

## Code Review

Copilot code review provides AI-powered pull request reviews on GitHub.com.

### Custom Review Instructions

Review behavior can be customized with instruction files:

| File | Purpose |
|---|---|
| `.github/copilot-instructions.md` | General instructions that also apply to code review |
| `.github/instructions/*.instructions.md` | Path-specific review instructions (use `applyTo` globs) |

```markdown .github/instructions/security.instructions.md
---
applyTo: "src/auth/**/*.ts"
---
When performing a code review on authentication code:
- Verify all inputs are sanitized
- Check for SQL injection vulnerabilities
- Ensure tokens have appropriate expiration
- Flag any hardcoded credentials
```

<Note>
  `AGENTS.md` files are **not** currently supported for Copilot code review. Use `.github/copilot-instructions.md` and `.github/instructions/*.instructions.md` for review customization.
</Note>

## Agent Skills

Agent Skills are portable capability packages that extend Copilot with specialized knowledge and workflows. They work across Copilot in VS Code, Copilot CLI, and the Copilot coding agent.

### Skill Structure

```
.github/skills/
  deploy-preview/
    SKILL.md           # Skill instructions (required)
    templates/          # Optional bundled resources
      deploy-config.yaml
    scripts/
      verify-deploy.sh
```

```markdown .github/skills/deploy-preview/SKILL.md
---
name: "deploy-preview"
description: "Deploy a preview environment for the current branch"
---

## Deploy Preview

When asked to deploy a preview:

1. Read the deployment configuration from `templates/deploy-config.yaml`
2. Run the deploy script: `bash scripts/verify-deploy.sh`
3. Report the preview URL back to the user
```

### Progressive Loading

Skills use a three-level loading system for efficiency:

1. **Discovery**: Copilot reads the skill name and description from YAML frontmatter
2. **Instructions Loading**: The full SKILL.md body is loaded into context when relevant
3. **Resource Access**: Additional files in the skill directory are accessed on demand

### Storage Locations

| Scope | Location |
|---|---|
| Repository | `.github/skills/[skill-name]/SKILL.md` |
| Personal (global) | `~/.copilot/skills/[skill-name]/SKILL.md` |

## MCP Servers

GitHub Copilot supports the Model Context Protocol (MCP) for extending agent capabilities with external tools and data sources. MCP support in VS Code reached general availability in July 2025.

### Configuration

<Tabs>
  <Tab title="Workspace (.vscode/mcp.json)">
    Create `.vscode/mcp.json` in your project to share MCP server configurations with your team:

    ```json .vscode/mcp.json
    {
      "servers": {
        "github": {
          "type": "http",
          "url": "https://api.githubcopilot.com/mcp"
        },
        "playwright": {
          "command": "npx",
          "args": ["-y", "@microsoft/mcp-server-playwright"]
        },
        "my-api": {
          "command": "node",
          "args": ["./tools/mcp-server.js"],
          "env": {
            "API_KEY": "${input:my-api-key}"
          }
        }
      },
      "inputs": [
        {
          "id": "my-api-key",
          "type": "promptString",
          "description": "API key for my-api server",
          "password": true
        }
      ]
    }
    ```
  </Tab>
  <Tab title="User Profile">
    Run `MCP: Open User Configuration` from the VS Code Command Palette to configure MCP servers available across all workspaces.
  </Tab>
  <Tab title="Dev Containers">
    MCP servers can be configured in `devcontainer.json` for containerized development environments.
  </Tab>
</Tabs>

### Server Types

| Type | Description | Configuration |
|---|---|---|
| **stdio** | Local MCP servers communicating over standard I/O | `command` and `args` fields |
| **http** | Remote MCP servers using HTTP streaming | `url` field |

### GitHub MCP Server

GitHub provides an official MCP server available in two modes:

| Mode | Description |
|---|---|
| **Remote** (recommended) | Hosted by GitHub at `https://api.githubcopilot.com/mcp` |
| **Local** | Self-hosted on your machine for custom setups or specific security requirements |

The GitHub MCP server is available to all GitHub users regardless of plan. Individual tools inherit the access requirements of their corresponding GitHub features.

### MCP with Coding Agent

The Copilot coding agent can use tools from local and remote MCP servers. However, remote MCP servers that use OAuth for authentication are not currently supported by the coding agent.

<Warning>
  GitHub Copilot supports MCP **tools** but does not currently support MCP **resources**. Direct resource integration (as used in Claude Desktop) is not available.
</Warning>

### Enterprise MCP Policy

Enterprises and organizations can enable or disable MCP usage via the **"MCP servers in Copilot"** policy. The policy is disabled by default for enterprise/organization plans. Copilot Free, Pro, and Pro+ users are not governed by this policy.

## Hooks

Hooks are deterministic shell commands that run at strategic points in the agent workflow. They are configured as JSON files in `.github/hooks/`.

### Hook Types

| Hook | Trigger |
|---|---|
| `sessionStart` | When an agent session begins |
| `sessionEnd` | When an agent session ends |
| `userPromptSubmitted` | When the user sends a prompt |
| `preToolUse` | Before a tool is executed (can approve or deny) |
| `postToolUse` | After a tool completes |
| `agentStop` | When the agent stops |
| `subagentStop` | When a sub-agent stops |
| `errorOccurred` | When an error occurs |

```json .github/hooks/lint-check.json
{
  "event": "postToolUse",
  "command": "npm run lint -- --quiet",
  "description": "Run linter after code changes"
}
```

<Info>
  The `preToolUse` hook is the most powerful -- it can approve or deny tool executions before they happen, enabling hard policy gates at execution time.
</Info>

## Installation Provider Notes

This section covers what an AIPkg installer needs to know to programmatically install and configure packages for the GitHub Copilot platform.

### Configuration File Paths

| File | Path | Purpose |
|---|---|---|
| Repository instructions | `<repo>/.github/copilot-instructions.md` | Project-wide Copilot instructions |
| Path-specific instructions | `<repo>/.github/instructions/*.instructions.md` | Scoped instructions with `applyTo` globs |
| Custom agents | `<repo>/.github/agents/*.agent.md` | Agent persona definitions |
| Agent skills | `<repo>/.github/skills/[name]/SKILL.md` | Skill capability packages |
| Hooks | `<repo>/.github/hooks/*.json` | Agent lifecycle hooks |
| MCP servers (workspace) | `<repo>/.vscode/mcp.json` | Workspace-scoped MCP server configuration |
| MCP servers (user) | User profile `mcp.json` | User-scoped MCP server configuration |
| Personal instructions | `$HOME/.copilot/copilot-instructions.md` | Global personal instructions |
| Personal skills | `$HOME/.copilot/skills/[name]/SKILL.md` | Global personal skills |
| AGENTS.md | `<repo>/AGENTS.md` | Cross-agent instructions (also read by Copilot) |

### Installing MCP Server Extensions

To programmatically register an MCP server, merge the server definition into `.vscode/mcp.json`:

```json
{
  "servers": {
    "package-server-name": {
      "command": "npx",
      "args": ["-y", "@scope/mcp-server-package"],
      "env": {
        "SOME_API_KEY": "${input:some-api-key}"
      }
    }
  },
  "inputs": [
    {
      "id": "some-api-key",
      "type": "promptString",
      "description": "API key for the MCP server",
      "password": true
    }
  ]
}
```

**Key considerations:**

- The `servers` object is a flat map; each key is a unique server identifier
- Merge carefully -- do not overwrite existing server entries
- Use `inputs` for secrets to avoid hardcoding API keys
- VS Code provides IntelliSense for the `mcp.json` schema
- For remote servers, use `"type": "http"` with a `"url"` field instead of `command`/`args`
- Include `.vscode/mcp.json` in source control to share configurations with the team

### Installing Custom Instructions

To install project-level instructions, write to `.github/copilot-instructions.md`. If the file already exists, append to it.

For path-specific instructions, create a new `*.instructions.md` file in `.github/instructions/` with appropriate `applyTo` frontmatter. Each file is independent, so adding a new file does not conflict with existing ones.

<Note>
  Unlike Goose's `.goosehints`, Copilot instruction files use Markdown format with optional YAML frontmatter. The `applyTo` field supports glob patterns (e.g., `"src/**/*.ts"`). All matching instruction files are combined -- there is no override behavior.
</Note>

### Installing Agent Skills

To install a skill, create a directory under `.github/skills/[skill-name]/` containing at minimum a `SKILL.md` file with YAML frontmatter (`name` and `description` fields). Additional resources (templates, scripts, examples) can be placed alongside it.

### Installing Hooks

To install a hook, create a JSON file in `.github/hooks/` with an `event` field specifying the lifecycle event and a `command` field with the shell command to execute.

### Config File Formats

| File | Format | Merge Strategy |
|---|---|---|
| `copilot-instructions.md` | Markdown | Append to existing content |
| `*.instructions.md` | Markdown + YAML frontmatter | Add new files (no merge needed) |
| `mcp.json` | JSON | Deep-merge `servers` and `inputs` objects |
| `*.agent.md` | Markdown + YAML frontmatter | Add new files (no merge needed) |
| `SKILL.md` | Markdown + YAML frontmatter | One per skill directory |
| `*.json` (hooks) | JSON | Add new files (no merge needed) |

### VS Code Extension Installation

To programmatically install the Copilot extension:

```bash
# Install via VS Code CLI
code --install-extension GitHub.copilot

# Verify installation
code --list-extensions | grep GitHub.copilot
```

### Non-Interactive Considerations

- Copilot requires GitHub authentication, which typically involves an interactive browser-based OAuth flow
- For CI/CD or automated environments, use the Copilot coding agent (assign issues on GitHub.com) or the GitHub CLI with `gh copilot`
- MCP server configurations in `.vscode/mcp.json` take effect when VS Code opens the workspace -- no restart is needed
- Instruction files (`.github/copilot-instructions.md`, `.github/instructions/*.instructions.md`) are picked up automatically with no restart required
